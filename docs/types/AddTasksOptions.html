    Type Alias AddTasksOptions

    Fine-grained options controlling how tasks are added to a Redis Stream using addTasks.

    AddTasksOptions allows you to tune trimming behavior, idempotency, batching and basic progress-tracking for bulk task insertion.

    Every option is optional - defaults are designed to be safe and predictable.

    Stream trimming & retention

    Example

    await queue.addTasks('queue:emails', data, {
      maxlen: 200000,
      approx: true,
      minidWindowMs: 300000, // keep only last 5 minutes
      idem: true,
      status: true,
    });

    idem?: boolean

    Whether to attach an idempotency key to each task when using object payloads. When enabled:

    • the worker ensures each key is processed at most once,
    • retries or duplicates with the same key are skipped automatically.
